# IAM 存取
{: #userroles}

在您帳戶中，組織成資源群組的所有服務都是使用 {{site.data.keyword.Bluemix_notm}} Identity and Access Management (IAM) 進行管理。帳戶擁有者會自動獲指派 Cloud IAM 的帳戶管理者角色。身為帳戶管理者，您可以藉由指派 Cloud IAM 角色來指派和管理使用者的存取權、建立資源群組，以及佈建服務實例。您可以藉由建立原則來提供使用者及服務 ID 的存取權，而這些原則設定使用者或服務 ID 要存取的目標，以及設定可定義所容許存取類型的角色。


## 何謂 Cloud IAM 原則以及誰可以指派它們？
{: #iamusermanpol}

原則會將對一組資源的一個或多個角色授與給使用者，以便能在指定的目標資源環境定義內採取特定的動作。當您指派原則時，請先選擇設定資源群組或個別資源的原則。然後，根據您的起始選擇，您可以選取資源群組內的服務或所選取資源的單一實例。根據您選取的服務，可能還有其他配置選項可供使用。最終，您可以選取要指派的一或數個角色。 

如果您有適當的角色，可以指派及管理原則。下表顯示原則管理作業，以及各項作業所需的角色。

|動作|所需的角色|
|----------|---------|
|建立帳戶中所有服務及實例的原則|帳戶擁有者，或者帳戶中所有服務的管理者| 
|建立帳戶中之服務的原則|帳戶擁有者，或者帳戶中服務的管理者|
|建立服務實例的原則|帳戶擁有者、帳戶中服務的管理者、相關資源群組中所有服務的管理者，或服務實例的管理者|
|建立用來管理資源群組的原則|帳戶擁有者或資源群組的管理者|
{: caption="表 1. 容許建立存取原則的使用者" caption-side="top"} 


## Cloud IAM 角色
{: #iamusermanrol}

使用 Cloud IAM，您可以管理及定義帳戶中使用者和資源的存取權。可以指派兩種角色：平台管理角色及服務存取角色。

<dl>
<dt>平台管理角色</dt> 
<dd>平台管理角色涵蓋某範圍的動作，包括建立實例、管理服務 ID、管理使用者和許可權以及建立資源群組的能力。最常見的平台角色是管理者、編輯者、操作員、檢視者。</dd>
<dt>服務存取角色</dt>
<dd>服務存取角色定義使用者或服務對服務實例執行動作（例如存取使用者介面或執行 API 呼叫）的能力。有三種可能的角色：管理員、撰寫者和讀者。</dd>
</dl> 

在使用者介面中指派原則時，您可能不會看到所有角色都列為選項，因為只會顯示適用於您選取之服務的角色。如需哪些角色已啟用以及每個存取角色針對每個服務允許哪些動作的特定資訊，請參閱該服務的文件。
{: tip}

在單一存取原則中使用這些角色的組合，您可以藉由對下列任一項指派存取權，來提供使用者及服務 ID 的精細存取權：

* 帳戶內的所有資源
* 所有服務內屬於個別資源群組的所有資源，以及管理資源群組的能力
* 資源群組中單一服務內的所有資源，以及管理資源群組的能力
* 帳戶中單一服務內的所有資源，不論獲指派它們的資源群組為何
* 個別實例中的資源
* 實例內的單一資源類型（例如，{{site.data.keyword.objectstorageshort}} 實例中的儲存區）

若要讓另一位使用者完整存取帳戶，以管理使用者存取權以及管理所有帳戶資源（包括建立資源群組的能力），請設定原則，讓使用者存取帳戶中的所有資源，方法是選取已指派**管理者**角色的**所有已啟用身分及存取的服務**。
{: tip}

### 平台管理角色

平台管理角色讓使用者能被指派不同的許可權層次，以在帳戶內執行平台動作。下表提供獲指派各角色之使用者可以執行的部分平台管理動作範例。您可以參閱每一個服務的文件，以瞭解如何將角色套用至所使用之服務環境定義內的使用者。

|存取原則詳細資料|使用者可以對帳戶中的服務執行的動作|服務 ID 的動作|存取資源群組的動作|資源群組內資源的動作|管理存取群組的動作|
|:-----------------|:--------------|:---------------|:----------------|:-----------------|:--------------|
| *服務* |選取的一個或所有已啟用 IAM 的服務|選取的 IAM 身分服務|選取的資源群組|資源群組中的選取服務|選取的 IAM 存取群組|
|檢視者角色|檢視實例、別名、連結及認證|檢視 ID 及 API 金鑰|檢視資源群組|僅檢視資源群組中指定的實例|檢視存取群組及成員|
|操作員角色|檢視實例，並管理別名、連結及認證|不適用|不適用|不適用|不適用|
|編輯者角色|建立、刪除、編輯及檢視實例。管理別名、連結及認證|建立及刪除 ID 和 API 金鑰|檢視及編輯資源群組的名稱|僅建立、刪除、編輯、暫停、回復、檢視及連結資源群組中指定的實例|檢視、建立、刪除及編輯帳戶中的存取群組|
|管理者角色|服務的所有管理動作|建立及刪除 ID 與 API 金鑰、指派原則給 ID|檢視、編輯及管理資源群組的存取權|資源群組中指定實例的所有管理動作|檢視、建立、刪除、編輯及管理使用存取群組的存取權|
{: caption="表 2. 平台管理角色及動作範例" caption-side="top"}
{: #platformrolestable}

部分服務可能將特定的動作對映至與服務管理相關的平台管理角色，而不是與服務存取相關的平台管理角色。例如，請參閱下表，下表詳述對映至這些角色的 {{site.data.keyword.containershort_notm}} 服務動作。


|平台管理角色|動作的說明|{{site.data.keyword.containershort_notm}} 的動作範例|
|:-----------------|:-----------------|:-----------------|
|檢視者|可以檢視服務實例，但不能修改| <ul><li>列出叢集</li><li>檢視叢集的詳細資料</li></ul>|
|編輯者|執行所有平台動作，但管理帳戶和指派存取原則除外|<ul><li>將服務連結至叢集</li><li>建立 Webhook</li></ul> |
|操作員|執行配置及操作服務實例所需的平台動作，例如檢視服務的儀表板。| <ul><li>新增或移除工作者節點</li><li>重新開機或重新載入工作者節點</li><li>將服務連結至叢集</li></ul> |
|管理者|根據指派給這個角色的資源執行所有平台動作，包括指派存取原則給其他使用者。|<ul><li>移除叢集</li><li>建立叢集</li><li>更新使用者存取原則</li><li>檢視者、編輯者及操作員可以執行的所有動作</li></ul>|
{: caption="表 3. {{site.data.keyword.containershort_notm}} 服務的平台管理角色及動作範例" caption-side="top"}


### 服務存取角色

服務存取角色讓使用者能被指派不同的許可權層次，以呼叫服務的 API 以及存取服務的使用者介面。下表提供可以根據指派的角色且根據使用 {{site.data.keyword.objectstorageshort}} 服務採取的範例動作。

**附註**：可以針對每個已指派角色採取的動作，會視您為原則選取的服務而不同。

|服務存取角色|動作的說明|{{site.data.keyword.objectstorageshort}} 服務的動作範例|
|:-----------------|:-----------------|:-----------------|
|讀者|在服務內執行唯讀動作，例如檢視服務特有的資源|列出及下載物件|
|撰寫者|撰寫者具有超越讀者角色的許可權，包括建立及編輯服務特有的資源。|建立及破壞儲存區與物件|
|管理員|管理員具有超越撰寫者角色的許可權，以完成服務所定義的特許動作。此外，您也可以建立及編輯服務特有的資源。|管理儲存空間的所有層面、建立及破壞儲存區與物件|
{: caption="表 4. 服務存取使用者角色及動作範例" caption-side="top"}
